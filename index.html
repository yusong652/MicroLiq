Research

From particle contacts to earthquake response

My research traces a single thread: understanding why soil loses strength during earthquakes, at the scale of individual particle contacts. Along the way, I developed new simulation methods, validated them experimentally, and built AI tools to accelerate the entire workflow.

01

Microscopic Fabric Analysis

Investigated fabric evolution in granular soils through true triaxial DEM simulations under general stress conditions, including various stress paths and Lode angles.

Details Han et al. (2023) KSCE J Civ Eng
02

DEM-Based HCA Simulation

Developed a coupled analytical servo mechanism that simultaneously satisfies undrained conditions and complex stress states in hollow cylinder apparatus simulations, enabling DEM-based cyclic torsional shear under arbitrary stress conditions.

Details Ma, Han et al. (2024) Comput Geotech
03

Experimental Validation

Conducted 16 undrained cyclic torsional shear tests on Toyoura sand using a hollow cylinder apparatus across K0 = 0.20–3.0. Validated the DEM-predicted non-monotonic relationship between K0 and liquefaction resistance with physical evidence.

Test data
04

Multi-Directional Seismic Loading

Proposed the double-8 loading path to quantitatively isolate multi-directional shear effects on liquefaction, maintaining equal shear stress magnitudes across different loading patterns.

Animations
05

AI-Powered DEM Workflow

Bridging the gap between LLM agents and domain-specific numerical simulation. Built an AI agent platform that connects to PFC through real-time WebSocket, and extracted the documentation/execution layer into a standalone MCP server usable with any compatible client.

Featured Project

pfc-mcp

An open-source Model Context Protocol server that brings LLM-powered workflows to ITASCA PFC. Born from years of working with PFC's domain-specific command language, this tool solves a real pain point: bridging the gap between modern AI assistants and specialized numerical simulation software.

Documentation Intelligence Browse and search PFC command docs, Python API, and reference materials — contact models, range elements — through structured tools
Task Execution Bridge Submit Python scripts to a running PFC instance via WebSocket, monitor progress, and capture plot output — all from any MCP client
Universal Compatibility Works with Claude Code, Claude Desktop, VS Code, or any MCP-compatible client. No vendor lock-in

Why it matters

PFC uses a specialized command syntax poorly represented in LLM training data. This server provides structured, verified documentation — eliminating hallucination and enabling reliable AI-assisted simulation workflows.
